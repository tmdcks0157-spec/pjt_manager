# Plan: 다크모드 + 설정 페이지

## Executive Summary

| 관점 | 내용 |
|------|------|
| **Problem** | 앱이 라이트 모드만 지원해 어두운 환경에서 눈 피로가 크고, 사용자 환경설정(시작 페이지 등)을 바꿀 방법이 없다 |
| **Solution** | `/settings` 페이지 신설 + Tailwind `dark:` class 전략으로 다크모드 구현, `localStorage`에 테마·시작 페이지 저장 |
| **Function UX Effect** | 설정 한 곳에서 테마·시작 페이지 변경 가능, 새로고침 후에도 설정 유지, 다크 환경에서 눈 편한 UI 제공 |
| **Core Value** | "내 방식대로 쓰는 개인 도구" — 불필요한 설정 없이 꼭 필요한 환경 커스터마이징 |

## 2. 기능 범위

### 2.1 설정 페이지 (`/settings`)

| 설정 항목 | 옵션 | 저장 위치 |
|-----------|------|-----------|
| 테마 | 라이트 / 다크 / 시스템(OS) | `localStorage['theme']` |
| 시작 페이지 | 오늘 / 프로젝트 / 전체현황 / 캘린더 / 주간리포트 | `localStorage['start-page']` |

> 계정 정보(이메일) 표시는 읽기 전용으로 포함. 비밀번호 변경 등은 Out of Scope.

### 2.2 다크모드 적용 범위

Tailwind CSS v4의 `dark:` variant를 사용. `<html>` 태그에 `dark` 클래스를 토글하는 방식.

**적용 대상 (이번 구현):**

| 영역 | 파일 |
|------|------|
| 사이드바 | `(main)/layout.tsx` |
| 칸반 페이지 전체 | `projects/[id]/page.tsx` |
| 대시보드 | `dashboard/page.tsx` |
| 오늘 페이지 | `today/page.tsx` |
| 전체 현황 | `overview/page.tsx` |
| 주간 리포트 | `report/page.tsx` |
| 캘린더 | `calendar/page.tsx` |
| 로그인/회원가입 | `(auth)/login`, `(auth)/register` |
| 설정 페이지 | `settings/page.tsx` (신규) |

### 2.3 Out of Scope

- 비밀번호 변경, 프로필 사진 업로드
- 알림 설정
- 언어 설정

## 3. 기술 설계

### 3.1 테마 시스템 구조

```
ThemeProvider (클라이언트 컴포넌트)
  └── 마운트 시 localStorage['theme'] 읽기
  └── 'dark' → <html class="dark">
  └── 'system' → prefers-color-scheme 감지
  └── 'light' (기본) → class 없음

useTheme hook
  └── currentTheme 반환
  └── setTheme(theme) → localStorage 저장 + html class 업데이트
```

### 3.2 Tailwind v4 다크모드 설정

Tailwind CSS v4는 `tailwind.config.ts`가 없는 CSS-first 방식.
`globals.css`에서 `@variant dark (.dark &)` 설정으로 class 전략 적용.

```css
/* globals.css */
@import "tailwindcss";
@variant dark (&:where(.dark, .dark *));
```

### 3.3 localStorage 키 규약

| 키 | 값 | 기본값 |
|----|-----|--------|
| `theme` | `'light'` \| `'dark'` \| `'system'` | `'light'` |
| `start-page` | `/today` \| `/dashboard` \| `/overview` \| `/calendar` \| `/report` | `/today` |

### 3.4 파일 구조 (신규 생성)

```
src/
  providers/
    ThemeProvider.tsx      ← 기존 QueryProvider와 동일 패턴
  hooks/
    useTheme.ts            ← 테마 상태 읽기/쓰기 훅
  app/
    (main)/
      settings/
        page.tsx           ← 설정 페이지
```

## 4. 사용자 스토리

| ID | 스토리 | 우선순위 |
|----|--------|---------|
| US-01 | 사용자는 설정 페이지에서 다크/라이트/시스템 테마를 선택할 수 있다 | P0 |
| US-02 | 선택한 테마는 새로고침 후에도 유지된다 | P0 |
| US-03 | 사용자는 로그인 후 이동할 시작 페이지를 설정할 수 있다 | P0 |
| US-04 | 사이드바에서 설정 페이지로 바로 접근할 수 있다 | P0 |
| US-05 | 시스템 테마 선택 시 OS 다크모드 설정을 자동으로 따른다 | P1 |

## 5. 구현 순서

```
Step 1. Tailwind v4 dark variant 설정 (globals.css)
Step 2. ThemeProvider 컴포넌트 + useTheme 훅 생성
Step 3. root layout에 ThemeProvider 추가
Step 4. /settings 페이지 생성 (테마 + 시작 페이지 설정 UI)
Step 5. 사이드바에 설정 링크 추가
Step 6. 각 페이지/컴포넌트에 dark: 클래스 적용
         └── 사이드바 → 칸반 → 대시보드 → 나머지 페이지 순
```
