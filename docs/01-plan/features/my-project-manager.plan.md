## 1. 개요

### 1.1 배경 및 목적

개인 개발자/기획자가 여러 도구를 오가며 프로젝트를 관리하는 비효율을 해소하기 위해,
태스크 관리 · 칸반 보드 · 일정 캘린더를 하나로 통합한 **개인용 프로젝트 관리 웹앱**을 개발한다.

### 1.2 목표

- 프로젝트와 태스크를 직관적으로 생성/관리
- 칸반 보드로 진행 상태를 시각화
- 캘린더로 마감일 기반 일정 관리
- Notion 감성의 깔끔한 UI/UX

## 2. 사용자 스토리

### 2.1 핵심 스토리

| ID | 스토리 | 우선순위 |
|----|--------|---------|
| US-01 | 사용자는 프로젝트를 생성하고 이름, 색상, 설명을 설정할 수 있다 | P0 |
| US-02 | 사용자는 프로젝트 내에 태스크를 생성하고 제목, 설명, 마감일을 입력할 수 있다 | P0 |
| US-03 | 사용자는 태스크를 칸반 보드(할 일 / 진행 중 / 완료)에서 드래그&드롭으로 이동할 수 있다 | P0 |
| US-04 | 사용자는 캘린더 뷰에서 마감일 기준으로 태스크를 확인할 수 있다 | P0 |
| US-05 | 사용자는 이메일/비밀번호로 로그인/회원가입하여 데이터를 클라우드에 저장하고 어디서든 접근할 수 있다 | P0 |
| US-06 | 사용자는 태스크에 우선순위(낮음/보통/높음/긴급)를 설정할 수 있다 | P1 |
| US-07 | 사용자는 태스크에 태그를 붙이고 태그로 필터링할 수 있다 | P1 |
| US-08 | 사용자는 대시보드에서 전체 프로젝트 진행 현황을 한눈에 볼 수 있다 | P1 |
| US-09 | 사용자는 태스크에 메모/노트를 작성할 수 있다 | P2 |
| US-10 | 사용자는 완료된 태스크 통계(주간/월간)를 확인할 수 있다 | P2 |

## 3. 기능 범위

### 3.1 In Scope (구현 대상)

**P0 - 핵심 기능 (MVP)**
- [ ] 회원가입 / 로그인 (이메일 + 비밀번호)
- [ ] 프로젝트 CRUD (생성, 조회, 수정, 삭제)
- [ ] 태스크 CRUD (제목, 설명, 마감일)
- [ ] 칸반 보드 (3단계 상태: 할 일 / 진행 중 / 완료)
- [ ] 드래그&드롭 상태 변경
- [ ] 캘린더 뷰 (월간)

**P1 - 향상 기능**
- [ ] 태스크 우선순위 설정
- [ ] 태그 시스템 + 필터링
- [ ] 대시보드 (프로젝트별 진행률)
- [ ] 다크모드

**P2 - 추가 기능**
- [ ] 태스크 내 메모/노트
- [ ] 완료 통계 차트

### 3.2 Out of Scope (제외)

- 팀 협업 기능 (멤버 초대, 권한 관리)
- 파일 첨부
- 알림/푸시
- 모바일 앱
- AI 기능

## 4. 기술 스택

```
Frontend:
- Next.js 14 (App Router)
- TypeScript
- Tailwind CSS
- TanStack Query (서버 상태)
- Zustand (클라이언트 상태)
- @dnd-kit (드래그&드롭)
- FullCalendar 또는 react-big-calendar (캘린더)

Backend (BaaS):
- Supabase
  - 인증 (이메일/비밀번호)
  - 데이터베이스 (PostgreSQL)
  - REST API 자동 생성

배포:
- Vercel (프론트엔드)
- bkend.ai (백엔드)
```

## 5. 데이터 모델 (초안)

```
Project {
  _id: string
  userId: string         // 소유자
  name: string
  description: string
  color: string          // 프로젝트 색상 (#hex)
  createdAt: Date
  updatedAt: Date
}

Task {
  _id: string
  projectId: string      // 속한 프로젝트
  userId: string
  title: string
  description: string
  status: 'todo' | 'in_progress' | 'done'
  priority: 'low' | 'normal' | 'high' | 'urgent'
  dueDate: Date | null
  tags: string[]
  notes: string
  order: number          // 칸반 보드 내 순서
  createdAt: Date
  updatedAt: Date
}
```

## 6. 화면 구성 (페이지 목록)
