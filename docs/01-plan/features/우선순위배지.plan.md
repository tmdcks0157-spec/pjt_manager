# Plan: 우선순위배지

## Executive Summary

| 관점 | 내용 |
|------|------|
| **Problem** | 대시보드에서 긴급/높음 태스크가 얼마나 쌓였는지 한눈에 알 수 없고, 칸반 컬럼 내 태스크가 우선순위 무관하게 나열된다 |
| **Solution** | 대시보드에 긴급/높음 통계 카드 추가, 칸반 컬럼 헤더에 우선순위 정렬 버튼 추가 |
| **Function UX Effect** | 앱 진입 즉시 "지금 급한 것"을 확인, 긴급 태스크를 자동으로 상단에 배치 |
| **Core Value** | "흩어진 일을 한 곳에서 놓치지 않고 끝낸다" |

## 2. 기능 요구사항

### FR-01: 대시보드 우선순위 통계 카드
- 상단 통계 카드 영역에 "긴급" 카드 추가 (urgent 태스크 수)
- "높음" 카드 추가 (high 태스크 수)
- 클릭 시 해당 우선순위 태스크 목록 하이라이트 또는 이동 (v2 고려)
- 삭제/보관된 태스크 제외 (deleted_at IS NULL, archived = false)

### FR-02: 프로젝트 카드 긴급 뱃지
- 프로젝트 카드에 urgent 태스크가 1개 이상이면 빨간 뱃지 표시
- 뱃지 내용: `긴급 N`

### FR-03: 칸반 컬럼 우선순위 정렬
- 컬럼 헤더 또는 필터 영역에 "우선순위 정렬" 토글 버튼 추가
- 활성화 시: urgent → high → normal → low 순 정렬
- 비활성화 시: 기존 order 값 기준 정렬 복원
- 정렬은 로컬 state (DB order 값 변경 없음)

---

## 3. 비기능 요구사항
- 대시보드 통계는 기존 Supabase 쿼리에 priority 집계 추가 (별도 쿼리 최소화)
- 우선순위 정렬은 클라이언트 사이드 (성능 영향 없음)
- 모바일 반응형 유지
