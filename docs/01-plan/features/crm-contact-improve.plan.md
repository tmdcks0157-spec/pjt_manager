# Plan: crm-contact-improve — 연락처 등록 개선

## Executive Summary

| 관점 | 내용 |
|------|------|
| **Problem** | ① 회사 패널에서 연락처를 바로 추가할 수 없어 별도 폼을 열어야 하고, ② 연락처가 많아지면 카드 뷰만으로는 한눈에 파악이 어려우며, ③ 이름 순 외 정렬 옵션이 없다 |
| **Solution** | 회사 패널에 "연락처 추가" 버튼 삽입(회사 자동 선택), 카드/리스트 뷰 토글 추가, 이름·최신·회사순 정렬 드롭다운 추가 |
| **Function UX Effect** | 회사 클릭 → 바로 그 회사 소속 연락처 추가, 리스트 뷰로 많은 연락처 스캔, 정렬로 원하는 기준 즉시 적용 |
| **Core Value** | 연락처가 많아질수록 빠르게 찾고 추가할 수 있는 효율적인 CRM 탐색 경험 |

---

## 1. 현황 분석

### 현재 CRM 구조 (`src/app/(main)/crm/page.tsx`)

| 항목 | 현재 상태 |
|------|-----------|
| 뷰 모드 | 카드 뷰만 존재 |
| 정렬 | `useContacts()` 반환값 그대로 (이름순, DB 쿼리 `.order('name')`) |
| 회사 패널 | 소속 연락처 목록 표시 O, 연락처 추가 버튼 X |
| 탭 | `contacts` / `companies` 탭 전환 |

### 재활용 가능한 것

| 항목 | 재활용 방법 |
|------|------------|
| `ContactForm` | `defaultCompanyId` prop 추가해 회사 자동 선택 |
| `ContactCard` 컴포넌트 | 카드 뷰 그대로 유지 |
| `filteredContacts` | 정렬 로직만 추가 |
| 회사 패널 JSX | 추가 버튼 1개 삽입 |

## 2. 기능 범위

### 2.1 회사에서 연락처 추가

| 항목 | 상세 |
|------|------|
| 위치 | 회사 사이드 패널 "소속 연락처" 헤더 우측 |
| 동작 | 클릭 시 ContactForm 오픈, `company_id` 해당 회사로 자동 선택 |
| 구현 | `ContactForm`에 `defaultCompanyId?: string` prop 추가 |

### 2.2 리스트/카드 뷰 토글

| 항목 | 상세 |
|------|------|
| 위치 | 검색바 우측 아이콘 버튼 2개 (카드/리스트) |
| 카드 뷰 | 현재 `ContactCard` 그리드 유지 |
| 리스트 뷰 | 한 줄에 이름·회사·이메일·전화번호·태그·다음액션 표시 |
| 상태 저장 | `localStorage`에 `crm-view-mode` 저장 (새로고침 유지) |

### 2.3 정렬 기능

| 정렬 옵션 | 기준 |
|-----------|------|
| 이름순 (기본) | `name` 오름차순 |
| 최신순 | `created_at` 내림차순 |
| 회사순 | `company.name` 오름차순 (없으면 맨 뒤) |

- 위치: 검색바 우측 (뷰 토글 옆)
- UI: `<select>` 드롭다운
